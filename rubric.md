# Assignment Grading Rubric

## Criterion 1: Repository Structure & Organization (20%)

| Level | Description |
|-------|-------------|
| **Excellent** | Well-organized file structure, clear naming conventions, proper use of directories, `.gitignore` configured correctly |
| **Good** | Reasonable structure with minor organizational issues |
| **Satisfactory** | Basic structure present but disorganized or missing key elements |
| **Needs Improvement** | Poor organization, files scattered, no clear structure |

## Criterion 2: Code Quality & Best Practices (30%)

| Level | Description |
|-------|-------------|
| **Excellent** | Clean, readable code with consistent style, meaningful variable names, appropriate comments, no code smells |
| **Good** | Generally clean code with minor style inconsistencies |
| **Satisfactory** | Functional code but with readability or style issues |
| **Needs Improvement** | Difficult to read, poor naming, no comments, significant code smells |

## Criterion 3: Functionality & Requirements (30%)

| Level | Description |
|-------|-------------|
| **Excellent** | All requirements met, program works correctly, handles edge cases |
| **Good** | Most requirements met with minor issues |
| **Satisfactory** | Core functionality works but missing some requirements |
| **Needs Improvement** | Significant functionality missing or broken |

## Criterion 4: Documentation (20%)

| Level | Description |
|-------|-------------|
| **Excellent** | Comprehensive README with setup instructions, clear explanations, screenshots or examples where appropriate |
| **Good** | Good README covering most aspects |
| **Satisfactory** | Basic README present but lacking detail |
| **Needs Improvement** | No README or very minimal documentation |

---

**Scoring:** Each criterion is scored proportionally to its weight. The total score is the sum of all criteria scores scaled to the assignment's max points.

**Note:** Instructors should customize this rubric per assignment. Replace criteria and weights to match your specific assignment requirements.
